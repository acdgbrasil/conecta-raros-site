Key Findings
Conecta Raros has excellent metadata fundamentals — every page has unique titles, meta descriptions, complete Open Graph and Twitter Card tags, and proper canonical URLs. The homepage features strong structured data with Organization and FAQPage schema. However, the site suffers from 8+ broken placeholder links (href="#") in the footer across all pages, no favicon on any page, and thin content on the contact page (~200 words). The biggest strategic limitation is the site's size: with only 3 pages, it cannot build topical authority against competitors like raras.org.br (40+ institutions, 7,000+ words) and casahunter.org.br. The dashboard page does not link to the contact page, creating a conversion dead-end. This audit is limited to HTML analysis only — it cannot measure actual page speed, backlink profile, Core Web Vitals, or real search rankings.

Performance Indicators
Resource counts, image optimization, and loading patterns

External JavaScript from CDN Without Preconnect

All 3 pages load Phosphor Icons from unpkg.com via <script src="https://unpkg.com/@phosphor-icons/web"> without any preconnect hint. This adds a DNS lookup + TLS handshake delay.

Add <link rel="preconnect" href="https://unpkg.com" crossorigin> to all pages. Consider self-hosting the icon library for better performance control.
Medium

Images Use Legacy Formats (PNG, GIF)

Partner logos use PNG format (logo-roraima.png, logo-acdg.svg, logo-envolve.png). One animation uses GIF (inovacao.gif). No WebP or AVIF alternatives detected.

Convert PNG images to WebP with PNG fallback using <picture> element. Replace GIF animations with WebP or MP4 video.
Low

No Lazy Loading on Images

Partner logo images on index.html lack loading="lazy" attribute. While there are only 3 images, this is a best practice for Core Web Vitals.

Add loading="lazy" to images below the fold. Keep above-the-fold images eager-loaded.
